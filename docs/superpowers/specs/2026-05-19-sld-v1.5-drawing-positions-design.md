# SLD v1.5 + db-layout v1.3 — Drawing Layout (Stage 2) Design Spec

**Date:** 2026-05-19
**Skill targets:**
- `electrical/sld` v1.4.0 → v1.5.0
- `electrical/db-layout` v1.2.0 → v1.3.0 (paired — 4 new INT sub-DB examples to support the multi-sheet fixture)

**Sprint type:** Paired minor version bumps. Adds the *spatial-intent layer* to the SLD IR — engineering layout judgment (which boards group where, which CAD layer they belong on, when to split across drawing sheets) — while leaving geometric coordinate placement to the runtime renderer.

**Origin:** Part B Phase 2 of the SLD chapter per user direction 2026-05-18: "Close SLD chapter: v1.4 → v1.5 → then small-power". Deferred from SLD v1.3 spec §1 "future v1.5.0+ work" (`electrical/sld/docs/engineering-philosophy.md`).

**Skill/runtime boundary preserved:** Skills emit layout *intent* (hybrid structured fields + freeform note). Runtime project consumes intent + IR + symbol libraries → produces actual SVG/DXF/PDF. No rendering in this repo. See [[runtime-project-boundary]] memory.

---

## 1. Why this sprint

SLD v1.4 (shipped 2026-05-18) added multi-skill intent consumption — SLD now aggregates db-layout + earthing + fault-level intents into a logical topology with cross-skill provenance. But the IR remains coordinate-free: there is no `drawing_layout` block telling a downstream renderer which boards belong on which sheet, which CAD layer, or how they route.

v1.5 adds the spatial-intent layer. The LLM does engineering layout judgment (functional grouping, jurisdiction-appropriate drawing standards, multi-sheet split decisions for life-safety isolation); the runtime renderer does geometric layout execution (x/y coordinate placement, symbol bounding boxes, sheet pagination math).

**Critical architectural decision:** drawing_layout carries NO x/y coordinates. Skills do not compute geometry. Per [[runtime-project-boundary]], renderers live in the separate runtime project.

## 3. Architecture decisions

### 3.1 — `drawing_layout` is OPTIONAL in the schema

v1.3 and v1.4 examples remain valid without it. v1.5 examples ship it. Future versions may make it required (→ v2.0.0 breaking change). This matches the v1.4 backward-compatible pattern.

### 3.2 — Hybrid position vocabulary (4 fields per board)

Per board, the schema records:

| Field | Type | Purpose |
|---|---|---|
| `tree_layer` | int (0,1,2,…) | Topology depth (0 = root MSB) |
| `layout_group` | enum (8 values) | Functional grouping → drives CAD layer assignment |
| `routing_intent` | enum (5 values) | How the feeder reaches this board |
| `layout_intent_note` | string ≤200 chars | Engineer-reviewable rationale (the WHY) |

**Why hybrid:** pure spatial vocab (riser-relative) is UK-commercial-centred and breaks for industrial/substation. Pure grid is arbitrary. Pure tree is too sparse. Pure free-form is unparseable. Hybrid gives the LLM 3 structured enum picks + 1 narrative sentence — all in the LLM's wheelhouse — while runtime gets enough structure to render and engineers get reviewable English rationale.

### 3.3 — Layer naming via jurisdiction-aware lookup tables

The skill emits the abstract `layout_group` enum value. The concrete CAD layer name (e.g., `"E-FIRE-ALAR-MAIN"` for UK or `"E-FIRE-MAIN"` for US) is looked up at render time by the runtime, keyed by jurisdiction. This decouples engineering judgment from drawing-standard pedantry — a single SLD output can be rendered to BS 1192 or AIA without re-running the skill.

Three lookup tables ship in this sprint (BS 1192 for GB/INT, AIA for US, simplified IEC for INT/KE generic). KE specifically defers to KS 1700:2018 §313 routing (which routes to BS 7671 standards chain) — so KE uses the BS 1192 table.

### 3.4 — Multi-sheet via hybrid split rule

Hard count rule: SLDs with ≤8 boards default to single-sheet.

Soft functional rule (override): SLDs with `fire_alarm_life_safety` AND `general_power` boards coexisting MUST split sheets, even if total ≤8. This honors BS 9999 §6.4 / IEC 60364-5-56 §560 / NFPA 72 §10.6 life-safety isolation requirements.

Per-sheet `split_rationale` (string) documents the LLM's reasoning. Engineer can review.

### 3.5 — Implicit sheet boundaries (no explicit continuation refs)

Each board declares its `sheet_id`. When a feeder crosses sheets (e.g., MSB-MAIN on Sheet 1 feeds DB-FA1 on Sheet 2), the runtime renderer derives the continuation reference geometrically. The IR does NOT carry an explicit `sheet_continuation_refs[]` array — that's geometric layout, which is runtime work.

### 3.6 — Demonstration fixture: grow existing INT example

Rather than ship a new ~70-file hospital example, grow the existing `intl-commercial-msb-4subdbs` from 5 boards to 9 by adding 4 new sub-DBs (DB-EM, DB-COMMS, DB-UPS, DB-GENSET-XCV). This:

- Triggers BOTH multi-sheet rules (>8 boards + life-safety functional isolation)
- Reuses existing INT earthing + fault-level intents (peak_pfc + system_type cross-skill values unchanged)
- Requires only 4 new db-layout examples (not 12)
- Defers KE hospital scenario to v1.6 as a dedicated healthcare sprint

## 4. Schema shape — `drawing_layout`

Added as optional top-level field in `electrical/sld/schemas/sld-ir.schema.json`:

```jsonc
{
  "drawing_layout": {
    "type": "object",
    "required": ["sheets", "boards"],
    "additionalProperties": false,
    "properties": {
      "sheets": {
        "type": "array",
        "minItems": 1,
        "items": {
          "type": "object",
          "required": ["sheet_id", "sheet_size", "drawing_standard", "drawing_scale"],
          "additionalProperties": false,
          "properties": {
            "sheet_id":         { "type": "string", "pattern": "^sheet_[a-z0-9_]+$" },
            "sheet_size":       { "enum": ["A0", "A1", "A2", "A3", "Arch_E", "Arch_D", "Arch_C"] },
            "drawing_standard": { "type": "string" },
            "drawing_scale":    { "type": "string", "default": "NTS" },
            "split_rationale":  { "type": "string", "maxLength": 200 }
          }
        }
      },
      "boards": {
        "type": "object",
        "additionalProperties": {
          "type": "object",
          "required": ["sheet_id", "tree_layer", "layout_group", "routing_intent", "layout_intent_note"],
          "additionalProperties": false,
          "properties": {
            "sheet_id":           { "type": "string", "pattern": "^sheet_[a-z0-9_]+$" },
            "tree_layer":         { "type": "integer", "minimum": 0 },
            "layout_group":       { "enum": ["main", "general_power", "lighting", "mechanical",
                                             "fire_alarm_life_safety", "emergency_power",
                                             "comms", "other"] },
            "routing_intent":     { "enum": ["via_main_spine", "via_dedicated_riser",
                                             "via_shared_tray", "direct_from_msb",
                                             "via_genset_changeover"] },
            "layout_intent_note": { "type": "string", "maxLength": 200 }
          }
        }
      }
    }
  }
}
```

## 5. Worked example — INT growth to multi-sheet (9 boards, 2 sheets)

### 5.1 — Board roster after growth

Current `intl-commercial-msb-4subdbs` (5 boards):

```
MSB-MAIN + DB-L1 + DB-P1 + DB-M1 + DB-FA1
```

Grow to 9 boards by adding 4 sub-DBs:

| New board | layout_group | routing_intent | Function |
|---|---|---|---|
| DB-EM | `emergency_power` | `via_dedicated_riser` | EM lighting central battery cabinet per IEC 60364-5-56 §560 |
| DB-COMMS | `comms` | `via_dedicated_riser` | LV data/comms distribution per BS EN 50173 |
| DB-UPS | `emergency_power` | `via_dedicated_riser` | 10 kVA UPS-backed critical loads (30-min ride-through) |
| DB-GENSET-XCV | `emergency_power` | `via_genset_changeover` | ATS for standby-genset transfer |

### 5.2 — Multi-sheet split (per hybrid rule §3.4)

Trigger 1: total_boards = 9 (>8 hard count rule).
Trigger 2: `fire_alarm_life_safety` (DB-FA1) coexists with `general_power` (DB-L1, DB-P1) — functional isolation rule.

Result — 2 sheets:

```
Sheet 1 — General power + mechanical (4 boards):
  MSB-MAIN (main, intake)
  DB-L1    (general_power, lighting)
  DB-P1    (general_power, small power)
  DB-M1    (mechanical, HVAC)

Sheet 2 — Life-safety + comms + emergency (5 boards):
  DB-FA1        (fire_alarm_life_safety)
  DB-EM         (emergency_power, EM lighting central battery)
  DB-UPS        (emergency_power, UPS-backed)
  DB-COMMS      (comms)
  DB-GENSET-XCV (emergency_power, ATS)
```

Both sheets carry `sheet_size: "A1"`, `drawing_standard: "ISO 19650 + BS 1192:2007+A2:2016 (generic INT)"`, `drawing_scale: "NTS"`.

`split_rationale` per sheet:
- Sheet 1: `"General power group + mechanical — separated from life-safety per IEC 60364-5-56 §560"`
- Sheet 2: `"Life-safety + comms + emergency-power group — isolated from general power per BS 9999 §6.4 / IEC 60364-5-56 §560"`

### 5.3 — Cross-skill values preserved (no change from v1.4)

- `meta.consumed_intents[]` grows from 7 entries (5 db-layout + 1 earthing + 1 fault-level) to 11 entries (9 db-layout + 1 earthing + 1 fault-level)
- `distribution_hierarchy[]` grows from 5 to 9 entries
- `selectivity_cascade[]` grows from 4 to 8 entries (N-1 rule)
- `system_metrics.peak_pfc_ka` unchanged (still 22.5 kA from fault-level intent TX-1)
- INV-11 cross-skill checks still pass (system_type + peak_pfc tolerance)

## 6. Generator update — new Step 13

Inserted in `electrical/sld/prompts/generator.md` AFTER Step 12 (rationale). Authored last because drawing_layout depends on the complete IR (all boards and their roles) being available.

```markdown
## Step 13 — Author drawing_layout block (v1.5+)

drawing_layout is the SPATIAL-INTENT layer. The runtime renderer consumes it (with the IR + symbol library) to produce SVG/DXF/PDF. Skills do not compute x/y coordinates — that's runtime work.

### 13.A — Determine sheet count (hybrid split rule)

if total_boards ≤ 8 AND no (fire_alarm_life_safety AND general_power coexisting):
  → single sheet
else:
  → split:
    Sheet 1 = main + general_power + mechanical + lighting
    Sheet 2 = fire_alarm_life_safety + emergency_power + comms

### 13.B — Per sheet

Populate:
- sheet_id (kebab-case: "sheet_1", "sheet_2")
- sheet_size: jurisdiction default
    GB/INT/KE → "A1" (ISO 5457)
    US        → "Arch_D" (AIA)
- drawing_standard: jurisdiction default
    GB        → "BS 1192:2007+A2:2016"
    US        → "AIA CAD Layer Guidelines 2007"
    INT       → "ISO 19650 + BS 1192:2007+A2:2016 (generic INT)"
    KE        → "KS 1700:2018 §313 routes to BS 1192:2007+A2:2016"
- drawing_scale: "NTS" (SLDs are not-to-scale by convention)
- split_rationale: 1-sentence string explaining why these boards on this sheet (≤200 chars)

### 13.C — Per board

Populate:
- sheet_id (cross-reference)
- tree_layer (0 = root MSB)
- layout_group (from enum, per decision table below)
- routing_intent (from enum, based on how feeder physically reaches this board)
- layout_intent_note (≤200 chars, engineer-reviewable rationale)

Decision table for layout_group:
| Board role | layout_group |
|---|---|
| Root MSB | main |
| Lighting-only DB | lighting |
| General sockets/small-power DB | general_power |
| HVAC/mechanical DB | mechanical |
| Fire alarm DB | fire_alarm_life_safety |
| EM lighting / UPS-backed / genset-fed | emergency_power |
| Comms/data | comms |
| Anything else | other (rarely used) |

### 13.D — Life-safety isolation enforcement (SLD-internal)

Enforced from SLD-internal `layout_group` enum (no dependency on earthing intent flags):

If any board's `layout_group == "fire_alarm_life_safety"` AND any board's `layout_group == "general_power"`:
  → those boards MUST be on different sheets per BS 9999 §6.4 / IEC 60364-5-56 §560 / NFPA 72 §10.6
  → emit compliance flag `INV-14:life-safety-coexistence` if violated

Future versions (v1.6+) may add earthing-intent cross-skill checks (`supplementary_bonding_required` flag) when the earthing schema declares those flags. v1.5 enforces from SLD layout_group alone.
```

## 7. Validator update — INV-12, INV-13, INV-14

Appended to `electrical/sld/prompts/validator.md` after INV-11. All three fire only when `drawing_layout` is present (backward-compatible).

### INV-12: drawing_layout shape conformance

1. All `boards[*].sheet_id` reference an existing `sheets[].sheet_id` (no dangling refs)
2. All `boards.<id>` keys match `distribution_hierarchy[].board_id` (no orphans)
3. `tree_layer` ≥ 0
4. `layout_group` ∈ valid enum (8 values)
5. `routing_intent` ∈ valid enum (5 values)
6. `sheet_size` ∈ valid enum (7 values)

All hard fails.

### INV-13: jurisdictional sheet size

- jurisdiction ∈ {GB, EU, INT, KE} → `sheet_size` MUST be ISO (A0/A1/A2/A3)
- jurisdiction == US → `sheet_size` MUST be ANSI (Arch_E/D/C)

Warning flag only (engineer override valid if export covers both formats).

### INV-14: split logic conformance

1. if total_boards ≤ 8 AND no fire_alarm+general_power coexistence → exactly 1 sheet (hard fail if multi-sheet)
2. if total_boards > 8 → ≥2 sheets (hard fail if single-sheet)
3. if any `fire_alarm_life_safety` board AND any `general_power` board → MUST be on different sheets (hard fail)
