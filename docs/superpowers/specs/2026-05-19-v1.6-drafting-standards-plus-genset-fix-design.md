## 3. Architecture decisions

### 3.1 — Per-standard folder pattern mirrors `electrical/<standard>/`

Each of the 10 standards gets its own folder under `shared/standards/drafting/<STANDARD-CODE>/`. **Folder naming convention: alphanumeric only, no separators or punctuation, matching the published standard's short form.** Confirmed from existing 3 folders: `BS1192`, `ISO19650`, `AIA`. New folders follow the same convention: `ISO5457`, `ISO5455`, `ISO7200`, `ISO128`, `ISO129`, `ISO9431`, `BS5536`.

Each folder MUST contain:
- `meta.json` — publication info, edition, amendments, scope, structure (mirrors `electrical/BS7671/meta.json` shape)
- `README.md` — folder index + cross-references to other standards

Each folder SHOULD contain at clause-citation depth:
- 1+ engine-lookupable JSON files (data tables queryable by primary key — e.g., sheet_size → dimensions; status_code → meaning)
- 0+ supporting markdown files (narrative, terminology, security overview, etc.)

### 3.2 — Engine-lookupable JSON shape

Each data JSON follows the same structure as `electrical/BS7671/cable-current-ratings.json`:

```jsonc
{
  "_source": "<full standard name + clause/table reference>",
  "_clause": "<specific clause or table number>",
  "_year": "<edition + amendments>",
  "_note": "<engineer-facing guidance>",
  "<primary_key_name>": {  // e.g., "sheet_sizes", "status_codes", "scales"
    "<key>": { <data fields> }
  }
}
```

This shape supports runtime engine_lookup queries by primary key without parsing markdown.

### 3.3 — Top-level `shared/standards/drafting/meta.json` is the discovery entry point

Skills don't enumerate folders — they read the top-level meta.json's `standards{}` map + `jurisdiction_to_primary_standards{}` lookup. This is the only file SLD generator Step 13 or future skill prompts need to know about; everything else is dereferenced from there.

### 3.4 — ISO 19650 Parts 1-5 each as separate JSON files

ISO 19650 is a 5-part standard. Each part covers a distinct concern (concepts, delivery, operations, exchange, security). Separating them allows skills to consume specific parts (e.g., a future federation-aware skill might consume only Part 5 security data).

### 3.5 — BS 5536 as standalone UK companion (not folded into BS 1192)

BS 5536 is a UK reading of ISO 128 line conventions for technical drawings. It cross-references ISO 128 + BS 1192 but exists as a standalone published standard. Following the existing pattern of one-folder-per-published-standard, BS 5536 gets its own folder rather than being folded under BS 1192 or ISO 128.

### 3.6 — db-layout v1.3.1 is a tactical patch — no schema or example-count change

The genset Ik" fix is a single-example data correction. No new examples, no schema change, no contract change. Patch-level semver (v1.3.0 → v1.3.1) signals "non-breaking data accuracy fix" to consumers.

### 3.7 — SLD v1.5 examples unchanged

The expanded drafting standards layer DOES NOT trigger any update to existing SLD output.json files. SLD v1.5 references `drawing_standard` strings like `"BS 1192:2007+A2:2016"` which are now backed by a much deeper standards folder, but the LLM doesn't need to change its emission shape. Future v1.7+ consumer enhancements (e.g., adding `sheet_template_ref` to drawing_layout) are explicit out-of-scope.

### 3.8 — No drafting-standards-layer evals

Standards layers don't ship their own evals — skills consume them, and skill-level evals verify consumption correctness. v1.6 ships no new YAMLs in any `evals/` folder.

### Engineering correction (per IEC 60909-0:2016 §3.5.1)

**Calculation:**
- Genset: 80 kVA, 400 V TPN, salient-pole 4-pole machine
- In (full-load) = 80,000 / (√3 × 400) ≈ 115.5 A
- Subtransient reactance Xd" ≈ 0.12 pu (typical for 4-pole synchronous machine per IEC 60909-1:2002 Table A.1)
- Subtransient EMF E" ≈ 1.05 pu (per IEC 60909-0:2016 §3.5.1 equivalent voltage source)
- **Subtransient Ik" = In × E"/Xd" ≈ 115.5 × (1.05 / 0.12) ≈ 1.01 kA**
- Peak asymmetric ip ≈ κ × √2 × Ik" with κ ≈ 1.8 (low X/R ratio at genset terminals) → ip ≈ 2.57 kA
- Sustained Ik (5 s after fault, AVR field forcing settles) ≈ 0.4 kA (3-5 × In; consistent with the "3-4× rated" figure previously confused with Ik")

### Prose locations to correct (7 per reviewer)

Per SLD v1.5 final code review report:
- Line 44, 58, 65, 80, 106, 122, 125 in `output.json` rationale + assumptions
- Equivalent narrative paragraphs in `reasoning.md` (Genset Topology, ATS Transfer Logic, Compliance + Assumptions sections)
- `sample-schedule.md` ATS transfer-time disclosure footer if it cites genset-mode Ik"

### Correction template (use consistently)

**Old (wrong):**
> Genset-mode Ik" ≈ 3.5-4.5 kA at DB-GENSET-XCV...

**New (correct):**
> Genset-mode subtransient Ik" ≈ 1.0 kA (IEC 60909-0:2016 §3.5.1 equivalent voltage source with Xd" = 0.12 pu, E" = 1.05 pu); peak asymmetric ip ≈ 2.5 kA (κ ≈ 1.8). Sustained Ik (after AVR field forcing, ~5s post-fault) ≈ 0.4 kA (3-5× In). The earlier "3.5-4.5 kA" figure conflated sustained Ik with subtransient Ik".

## 8. Risks + mitigations

| Risk | Likelihood | Mitigation |
|---|---|---|
| Standards data sourcing accuracy — ISO 5457 sheet dimensions, BS 1192 layer codes, etc. need to be precise | Medium | Cite each clause + year qualifier in `_clause` / `_source` fields; mark uncertain values with `_unverified: true` flag for engineer review (same pattern as electrical standards) |
| ISO 19650 Parts 1-5 separately is verbose; Part 4 less commonly cited | Low | Match depth across Parts 1-3 + Part 5; Part 4 ships key concepts only (defer exhaustive exchange specs) |
| Cross-references between standards become a maintenance burden | Low | Add `_cross_refs` field per standard's meta.json (one direction only — referenced-from, not referenced-to) |
| Genset Ik" fix ripples to other examples | Low | Verify after fix: SLD examples consume the SLD-level peak_pfc_ka, not db-layout fault flags. Should be isolated. |
| ISO 9431 (drawing folds) is rarely cited in modern digital workflows | Low | Keep minimal (3 files including meta + README) |
| AIA depth varies — full taxonomy is 100+ codes per discipline | Medium | Focus on electrical major group + status; defer architectural/mechanical major groups to per-discipline-skill consumption sprints |
