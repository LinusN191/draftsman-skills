## 1. Why this sprint

The earthing skill v1.1.0 (shipped 2026-05-17) added a Kenya TN-S worked example and uses inline annotation `"BS 7671:2018+A2 Reg X.Y.Z (adopted by KS 1700)"` everywhere it cites Kenyan-jurisdiction rules. This is a workaround — there is no canonical KS 1700 content in the repo to point at. As soon as any KS-specific deviation matters (e.g., §411.3.3 universal socket-RCD policy, §722 EV charging absence), the inline annotation pattern breaks down.

The repo has 70+ standards layers built end-to-end as standalone folders (BS7671/, IEC60364/, NFPA70/, NFPA70E/, IEC61439/, etc.). KS 1700 should be authored the same way. Once it exists, the earthing skill stops doing annotation tricks and cites KS 1700 clauses directly the way it cites BS 7671 clauses.

Additionally, this sprint sets the template for the other African codes already stubbed in `local-codes/`: NIS 197 (Nigeria), SANS 10142 (South Africa). Each becomes a separate future sprint following this v1.2 pattern.

## 4. Authoring strategy — verification status

KS 1700:2018 PDF is not in this repo. Every KS1700/ file is marked:

```json
"verification_status": "draft-from-bs7671-derivative-needs-source-verification"
```

Every file gets a `_verification_note` explaining what was derived vs verified:

```
"_verification_note": "Drafted by structural derivation from BS 7671:2018+A2 (KS 1700:2018 adopts BS 7671 substantially per Annex E). Clause numbers and structural content match BS template; Kenya-specific deviations (§411.3.3 socket-RCD universality, §722 EV absence, climate-adjusted ambient temps) are sourced from KEBS technical bulletins and engineering practice in Kenya. Final verification against the published KS 1700:2018 PDF required before downstream skills rely on tight tolerances."
```

This mirrors `IEC60364/part5-52-cable-impedance.json` (shipped 2026-05-17 as `draft-from-iec-reference-needs-second-source-pair`). Honest signaling beats false precision. When the KS 1700:2018 PDF becomes available, a future micro-sprint promotes `verification_status` from draft to `verified-against-source` after a clause-by-clause check.

### KS-unique file schemas

**`annex-E-bs7671-adoption-table.json`** — KS↔BS adoption mapping:

```json
{
  "clause_ref": "KS 1700 2018:Annex-E",
  "_title": "KS 1700:2018 Annex E — Adoption-by-reference of BS 7671:2018",
  "verification_status": "draft-from-bs7671-derivative-needs-source-verification",
  "adoption_table": [
    {
      "ks_clause": "§411.3.2",
      "bs_clause": "BS 7671:2018+A2 Reg 411.3.2",
      "adoption_type": "verbatim",
      "_note": "Required disconnection times — Table 41.1 identical"
    },
    {
      "ks_clause": "§411.3.3",
      "bs_clause": "BS 7671:2018+A2 Reg 411.3.3",
      "adoption_type": "modified",
      "_note": "KS applies RCD universally on socket circuits ≤32A regardless of Zs; pre-AMD2 BS required only on specific circuit types. KS aligns with BS 7671 AMD 2 (2022)."
    },
    {
      "ks_clause": "§722",
      "bs_clause": "BS 7671:2018+A2 Part 7-722",
      "adoption_type": "not_adopted",
      "_note": "KS 1700:2018 has no §722. Annex E directs to IEC 60364-7-722:2018 for EV charging installations."
    }
  ]
}
```

`adoption_type` enum: `verbatim` | `modified` | `not_adopted` | `ks_specific`.

**`ks-unique-deviations.json`** — content with no BS analogue:

```json
{
  "deviations": [
    {
      "id": "socket-rcd-universal",
      "ks_clause": "§411.3.3",
      "summary": "30mA RCD additional protection mandatory on all socket circuits ≤32A regardless of Zs compliance",
      "rationale": "Kenya climate (humidity) and end-user environment risk profile justify universal RCD on socket circuits",
      "diverges_from": "BS 7671 pre-AMD2 socket-RCD selective rule",
      "engineering_impact": "Every Kenya design must include socket-RCD; engineers should not omit it even where Zs compliance is achieved"
    },
    {
      "id": "ambient-temperature-baseline",
      "ks_clause": "§522.1 (informative)",
      "summary": "Ambient design temperature baseline 30°C indoor / 40°C outdoor for Kenya climate zones",
      "rationale": "Coastal humidity + inland highland diurnal range require higher baselines than UK 25°C indoor / 20°C outdoor",
      "diverges_from": "BS 7671 Appendix 4 baseline 30°C",
      "engineering_impact": "Cable ampacity derating coefficient Ca applied more often; effective cable size larger for same load"
    }
  ]
}
```

### Generator prompt — jurisdiction routing logic

**Old (v1.1):**
```
GB → BS 7671 + IEC 60617
EU/INT → IEC 60364 + IEC 60617
KE → BS 7671 + IEC 60364 + "(adopted by KS 1700)" inline annotation
US → NFPA 70 + IEC 60617
```

**New (v1.2):**
```
GB → BS 7671 + IEC 60617
EU/INT → IEC 60364 + IEC 60617
KE → KS 1700 (primary) + IEC 60364 (fallback per KS Annex E §VIII) + IEC 60617
US → NFPA 70 + IEC 60617
```

### Citation form changes

**Old (v1.1):**
```
"code_clause": "BS 7671:2018+A2 Reg 411.3.3 (adopted by KS 1700)"
```

**New (v1.2):**
```
"code_clause": "KS 1700:2018 §411.3.3"
```

For clauses where KS adopts BS verbatim via Annex E, generator may explicitly note the adoption:
```
"code_clause": "KS 1700:2018 §411.4 (Annex E: adopts BS 7671:2018+A2 Reg 411.4 verbatim)"
```

But the primary citation form is always `"KS 1700:..."` (not `"BS 7671:..."` with annotation).

## 12. Risks + mitigations

| Risk | Likelihood | Mitigation |
|---|---|---|
| KS 1700:2018 PDF unavailable; content drift from published standard | High | Mark every file `verification_status: "draft-from-bs7671-derivative-needs-source-verification"`. Document the derivation source explicitly. Future micro-sprint promotes verification status once PDF available. |
| KS-specific deviations mis-stated | Medium | Sources documented per deviation in `ks-unique-deviations.json` `rationale` field. Engineer reading the file can flag issues. |
| Citation refactor breaks v1.1 KE example | Low | KE example is refreshed wholesale in Task 24. Schema unchanged (no IR shape break). |
| KS 1700 clause numbering differs from BS 7671 in places not yet known | Medium | KS Annex E adoption type `modified` flag captures these; explicit per-clause adoption_type prevents silent drift. |
| Other skills currently citing BS with KS-adoption annotation get out of sync | Low | Only the earthing skill currently uses the annotation pattern. Other skills using "BS 7671" citations stay valid (they're not KE-specific). Future skills can adopt the new pattern as they're updated. |
